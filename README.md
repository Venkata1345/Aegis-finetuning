## Summary

Most production PII detection is rule-based (Presidio): fast, free, predictable, but brittle on multilingual text, partial PII, OCR noise, and uncommon formats. Frontier APIs (GPT-4o, Gemini) hit much higher recall but charge per call and require sending sensitive text to a third party. **Aegis** fine-tunes a 7B open-weights model so you get frontier-level recall on a CPU/GPU you control, distributed as a small QLoRA adapter on top of Qwen 2.5 7B-Instruct.

> **Headline result.** Aegis tops the leaderboard on every test set: **0.954 strict F1** on the n=7,946 main test — ahead of GPT-4o-mini (0.892), Gemini Flash (0.885), base Qwen (0.612), and Presidio (0.438) — while running locally at **85ms p50 latency** (≈7× faster than the API baselines) at **zero per-call cost**. Trained on a single **NVIDIA A100 (Colab Pro)**, ~7.5 hours wall-clock, distributed as a 14 MB LoRA adapter.

- **Approach.** QLoRA (r=16, α=32, 4-bit base) on 300k [ai4privacy](https://huggingface.co/datasets/ai4privacy/pii-masking-300k) examples mapped to 9 PII categories: `PERSON`, `EMAIL`, `PHONE`, `ADDRESS`, `DOB`, `GOV_ID`, `FINANCIAL`, `MEDICAL_ID`, `IP_ADDRESS`. **Trained on a single NVIDIA A100 (Colab Pro) in ≈7.5 hours.** Adapter published at [Abhishek4545/aegis-qwen-7b-lora](https://huggingface.co/Abhishek4545/aegis-qwen-7b-lora).
- **Evaluation.** Three independent test sets — main (n=7,946 ai4privacy held-out split), adversarial (n=50 hand-written edge cases spanning 17 categories), and held-out (n=50 Wikipedia bio carriers with synthetic PII injected). Every metric in this README is produced by `eval/harness.py` from result JSONs; nothing is hand-edited.
- **Build order.** Eval-first — the metrics, runner, and harness were built and tested *before* training, so the comparison numbers are regenerable and the training delta over baselines is the headline result, not a self-graded summary.

## Metric definitions

All metrics operate on **character-offset spans** of the form `(type, start, end, text)`. Definitions match the implementations in [`eval/metrics.py`](eval/metrics.py).

- **Strict F1.** A predicted span counts as a true positive only when `(type, start, end)` matches a gold span exactly. Type-only mismatches at identical offsets count as *both* a per-type FN (gold's type) *and* a per-type FP (predicted type), so overall counts are the sum of per-type counts. Strict F1 is the demanding metric — it catches off-by-one boundary errors that partial F1 forgives.
- **Partial F1.** A predicted span counts when the type matches *and* the character-range IoU (intersection-over-union) ≥ 0.5. Each prediction matches at most one gold (greedy by best IoU); each gold matches at most one prediction; cross-type matches never count. Captures cases where the model finds the right entity with slightly off boundaries.
- **Schema valid.** Percentage of raw model outputs that parse cleanly through the Pydantic `SpanList` schema. Failures here mean the model produced invalid JSON or violated the array-of-spans shape — orthogonal to whether the spans are correct. Presidio is rule-based and always emits valid output; for LLM baselines this is a real measure of structured-output reliability.
- **Hallucination.** Percentage of *schema-valid* predicted spans where `text[start:end] != span.text`, or offsets are out of bounds. The model produced parseable JSON but invented offsets that don't anchor in the input. LLMs commonly score high here without offset-realignment post-processing — see [`baselines/base.py`](baselines/base.py)’s `realign_to_input` utility.
- **Correctly empty.** On adversarial cases with empty gold (`reference_no_value`, `zero_pii_control`, etc.), the percentage where the predictor also returned `[]`. F1 is undefined on no-PII inputs (no TP possible), so this is the dedicated metric for negative controls — it directly measures resistance to hallucinating spans on text that contains no PII.
- **Recall on injected (held-out only).** TP / (TP + FN) restricted to the 109 injected PII spans. Held-out precision is structurally biased because predictors correctly flag pre-existing Wikipedia PII (real DOBs, names of bio subjects) that the injection-only ground truth records as FP — so recall-on-injected is the meaningful number on this set.
- **p50 latency.** Median end-to-end inference time per request, measured by the harness ([`eval/perf.py`](eval/perf.py)). Includes prompt assembly, model inference, and output parsing — what a caller actually waits for.
- **$/1k calls.** Average API cost per 1,000 requests on this dataset, computed from token counts and per-model pricing in [`baselines/base.py`](baselines/base.py). `self` indicates self-hosted (no per-call cost; you pay for the GPU).

## Methodology

**Training data.** ai4privacy `pii-masking-300k`, English split, mapped from the source's privacy_mask labels into the 9-category Aegis schema via [`data/labels.py`](data/labels.py). Char-offset spans are taken directly from `privacy_mask` (no BIO conversion needed). Defensive checks reject empty text, bad/out-of-bounds offsets, value/text mismatches, and Pydantic-invalid spans — counts of each are reported by [`data/convert.py`](data/convert.py).

**Training run.** QLoRA on a single A100 (Colab Pro), 4-bit base + LoRA r=16, α=32, dropout 0.05; batched with Unsloth for memory efficiency. Final adapter restored from best-by-eval-loss with periodic Hub push. The full notebook is at [`train/train.ipynb`](train/train.ipynb).

**Eval sets.**
- **Main** — the ai4privacy held-out test split, n=7,946. In-distribution; the dispositive accuracy number.
- **Adversarial** — n=50 hand-written cases across 17 categories: phone-format variants, partial PII, embedded-in-code, multilingual (Chinese, Arabic, Cyrillic, Japanese, Hindi), near-miss negatives, OCR noise, long-context, density stress tests, and zero-PII controls. Built in [`data/adversarial.py`](data/adversarial.py); committed to [`data/eval/adversarial.jsonl`](data/eval/adversarial.jsonl).
- **Held-out** — n=50 Wikipedia first-paragraph bios with 1–4 synthetic PII items injected at random word boundaries via Faker. Carrier text is cached for deterministic re-runs. Built in [`data/heldout.py`](data/heldout.py).

**Calibration.** No human gold standard. Two independent LLM judges (GPT-4o + Gemini 2.5 Pro) score the same 50 adversarial outputs against gold using a calibrated 1–5 rubric; reported `r` is the Pearson correlation between them. Target r ≥ 0.6; below that, the rubric in [`eval/calibration.py`](eval/calibration.py) is iterated before any judge-based number is reported. This is a deliberate tradeoff for the portfolio context — documented openly rather than hidden.

## Limitations

- **Adversarial and held-out are n=50 each.** F1 estimates at this n have ~±0.14 width 95% CIs; the main test (n=7,946) is the dispositive accuracy number, and adversarial/held-out probe specific failure modes rather than overall accuracy.
- **Held-out precision is structurally biased.** Ground truth includes only injected PII; pre-existing Wikipedia PII (real DOBs, names of bio subjects) gets correctly flagged but counted as FP. Read **recall on injected**, not F1, on this set — it is the only metric that isn't confounded by the carriers' own PII.
- **Calibration uses cross-LLM-judge, not human gold.** Two judges agreeing isn't the same as being right; documented openly above. The rubric is in [`eval/calibration.py`](eval/calibration.py) and is iterated when r drops below 0.6.
- **Adversarial `ambiguity` category.** Cases like citing public-figure names in academic context have legitimately ambiguous ground truth; included for completeness but reported separately so they don't pollute the headline adversarial number.
